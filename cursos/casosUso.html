Casos de Uso

Descripción:

Un diagrama de casos de uso es una representación visual que describe las interacciones entre los actores (usuarios o sistemas externos) y un sistema en particular. Los casos de uso ayudan a capturar los requisitos funcionales del sistema y proporcionan una visión clara de cómo los actores interactúan con él.
A continuación se presentan las generaciones de los lenguajes de programación.

Actores

Los actores son entidades externas que interactúan con el sistema. Pueden ser usuarios reales, otros sistemas o dispositivos. Los actores se representan como figuras externas en el diagrama.

Casos de uso

Los casos de uso representan las diferentes acciones o funcionalidades que el sistema ofrece a los actores. Cada caso de uso describe una interacción específica entre un actor y el sistema. Se representan como óvalos en el diagrama.

Inclusión y extensión

Se utilizan relaciones de inclusión y extensión para mostrar cómo los casos de uso se relacionan entre sí. La inclusión indica que un caso de uso incluye la funcionalidad de otro caso de uso. La extensión indica que un caso de uso puede ampliar o modificar el comportamiento de otro caso de uso en situaciones específicas.

Interacciones entre actores y casos de uso

El diagrama de casos de uso muestra las interacciones entre los actores y los casos de uso mediante líneas conectivas. Estas líneas representan las acciones o intercambios de información entre el actor y el sistema durante la ejecución del caso de uso.

Documentación de requisitos

Los diagramas de casos de uso son una forma clara y concisa de documentar los requisitos funcionales del sistema. Sirven como base para la comunicación entre el equipo de desarrollo y los interesados en el proyecto.
