Entidad Relación

Descripción:

En una base de datos, una entidad relación es un modelo conceptual que representa las entidades (objetos o conceptos) y las relaciones entre ellas.
Los diagramas de ER se relacionan con los diagramas de estructura de datos (DSD), que se centran en las relaciones de los elementos dentro de las entidades, en lugar de las relaciones entre las entidades mismas.

Entidades

Las entidades representan objetos o conceptos del mundo real que tienen existencia independiente y atributos propios. Por ejemplo, en una base de datos de una tienda, las entidades pueden ser "cliente", "producto" y "pedido".

Relaciones

Las relaciones muestran cómo se conectan las entidades entre sí. Pueden ser de diferentes tipos, como uno a uno, uno a muchos o muchos a muchos. Por ejemplo, la relación entre "cliente" y "pedido" puede ser "uno a muchos", ya que un cliente puede realizar varios pedidos.

Atributos

Los atributos son características o propiedades de las entidades. Cada entidad tiene atributos específicos que describen sus características. Por ejemplo, el cliente puede tener atributos como el nombre, la dirección y el número de teléfono.

Cardinalidad

La cardinalidad define la cantidad de entidades con las que se relaciona otra entidad en una relación. Puede ser "uno", "muchos" o "opcional". Por ejemplo, en una relación "un cliente tiene muchos pedidos", la cardinalidad sería "uno a muchos".

Normalización

La normalización es un proceso utilizado para organizar los datos en una base de datos de manera eficiente y libre de redundancias. Se divide en diferentes formas normales (1NF, 2NF, 3NF, etc.) para garantizar que los datos estén estructurados de forma óptima y se eviten problemas como la duplicación de información.

Integridad referencial

La integridad referencial asegura que las relaciones entre entidades sean coherentes y válidas. Establece reglas que garantizan que no se puedan crear relaciones huérfanas (donde una entidad relacionada se elimina y la otra entidad queda sin conexión) o relaciones inconsistentes.
